使用Jenkins自动化打包并上传蒲公英

发表于 2017-03-01   |   分类于 安卓   |  

Jenkins是一个开源软件项目，旨在提供一个开放易用的软件平台，使软件的持续集成变成可能。

Jenkins是基于Java开发的一种持续集成工具，用于监控持续重复的工作，功能包括：持续的软件版本发布/测试项目；监控外部调用执行的工作。

Jenkins的安装

Tomcat下载

Jenkins的运行需要Tomcat服务器。
Tomcat官网：http://tomcat.apache.org/
Tomcat9下载：windows-x64 windows-x86

Jenkins下载

Jenkins官网：https://jenkins.io/index.html
Jenkins下载：点我下载

如何使用

把下载好的war包，放在tomcat的webapps下就可以运行。启动Tomcat，本地访问localhost:8080/jenkins即可进入jenkins页面

Jenkins的使用

启动

使用tomcat启动，进入tomcat下的/bin目录，启动tomcat

本地访问localhost:8080/jenkins即可进入jenkins页面

安装插件

jenkins支持多种插件，联网安装后即可使用，常见的有git插件，gradle插件，maven插件等等。
在jenkins主页左上角，点击系统管理–>管理插件

新版本的Jenkins首次使用时会默认弹出选择插件页面，有推荐设置和自定义设置，选择推荐设置自动下载常用插件即可。插件下载完毕即可进入Jenkins主页。

注：如遇到编码问题，可修改tomcat的server.xml配置，在Connector标签添加上URIEncoding=”UTF-8”

创建项目

选择新建，输入要构建的项目名称，选择第一个构建一个自由风格的软件项目。点击OK后，进入配置页面。
添加项目描述，以及选择项目来源，这里可以选择git或svn等。



输入仓库地址和分支名称

构建触发器

构建触发器有5中方案可选，这里使用第五种Poll SCM（固定时间检查，如果有更新，则构建，没有更新不构建），也可以同时使用多种构建方式。

每隔10分钟检查构建（若有更新）一次。

可参考：
Jenkins – 配置构建触发器
Jenkins之构建触发器配置

构建

构建中选择“Execute Windows Batch Command”，然后在输入框中输入要打包的命令即可。

邮箱设置

可以添加邮箱设置，构建失败会发送邮件给相应的邮箱地址，多个邮箱地址中间用空格隔开。

构建项目

配置完上述选项后就可以手动构建项目了，点击左侧面板的立即构建即可开始构建，下方build history会列出构建历史，红球代表构建失败，蓝球代表成功。

在build history中，点击构建记录，然后选择Console Output即可查看构建历史信息，出现错误可在此查看问题原因。
